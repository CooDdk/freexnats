# freexnats - NATS JetStream TUI 管理工具 - 设计文档

## 1. 项目概述

### 1.1 项目目标
开发一个基于 Bubble Tea TUI 框架的 NATS JetStream 管理工具 **freexnats**，提供专业、高效、炫酷的终端界面来管理 NATS JetStream 的各种资源。

### 1.2 品牌定位
- **项目名称**: freexnats
- **定位**: 免费、开源、高效的 NATS JetStream 终端管理工具
- **目标用户**: 开发人员、运维工程师、DevOps 工程师
- **设计风格**: 专业、现代、科技感、适当炫酷

### 1.3 技术栈
- **语言**: Go 1.25+
- **TUI 框架**: Bubble Tea (github.com/charmbracelet/bubbletea)
- **UI 组件**: Bubbles (github.com/charmbracelet/bubbles)
- **样式**: Lipgloss (github.com/charmbracelet/lipgloss)
- **NATS 客户端**: nats.go (github.com/nats-io/nats.go)

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 连接管理
- 配置 NATS 服务器连接参数(URL、认证信息)
- 建立和管理 NATS 连接
- 显示连接状态和服务器信息
- 支持断线重连

#### 2.1.2 Stream 管理
- **列表查看**: 显示所有 Stream 及其基本信息
- **详细信息**: 查看单个 Stream 的详细配置和统计信息
- **创建 Stream**: 通过表单创建新的 Stream
- **删除 Stream**: 删除指定的 Stream
- **清空 Stream**: 清空 Stream 中的消息
- **消息查看**: 浏览 Stream 中的消息

#### 2.1.3 Consumer 管理
- **列表查看**: 显示选定 Stream 下的所有 Consumer
- **详细信息**: 查看 Consumer 的详细配置和状态
- **创建 Consumer**: 通过表单创建新的 Consumer
- **删除 Consumer**: 删除指定的 Consumer
- **暂停/恢复 Consumer**: 控制 Consumer 的运行状态

#### 2.1.4 消息操作
- **发布消息**: 向指定 Stream 发布消息
- **查看消息**: 查看特定消息的详细内容
- **删除消息**: 删除 Stream 中的特定消息
- **消息搜索**: 根据条件搜索消息

#### 2.1.5 KV 存储管理
- **Bucket 管理**: 创建、删除、列出 KV Bucket
- **键值操作**: 查看、设置、删除键值对
- **历史记录**: 查看键的历史记录

#### 2.1.6 监控和统计
- **服务器信息**: 显示 NATS 服务器统计信息
- **健康检查**: 检查 JetStream 和服务器健康状态
- **实时监控**: 实时显示消息流量和资源使用情况

### 4.4 面向开发/运维人员的友好性设计

#### 4.4.1 设计原则
开发和运维人员的核心诉求是**高效、准确、可控**。工具设计遵循以下原则：
1. **键盘优先**：所有操作都可以通过键盘完成，鼠标操作作为补充
2. **符合肌肉记忆**：采用 Vim/Emacs/Shell 等常用工具的操作习惯
3. **信息密度高**：在有限的屏幕空间内展示尽可能多的有用信息
4. **操作可预测**：每个操作的结果清晰可见，支持撤销和确认
5. **快速定位**：提供强大的搜索、过滤和跳转功能
6. **可自动化**：支持脚本化和批量操作

#### 4.4.2 命令模式（类 Vim / Kubernetes 风格）
提供命令行模式，支持直接输入命令执行操作，提高操作效率。

**支持的命令示例：**
```
# Stream 操作
:stream list                    # 列出所有 Stream
:stream create <name> [opts]    # 创建 Stream
:stream delete <name>           # 删除 Stream
:stream info <name>             # 查看 Stream 详情
:stream purge <name>            # 清空 Stream
:stream subjects <name>         # 查看 Stream 的主题

# Consumer 操作
:consumer list <stream>         # 列出 Consumer
:consumer create <stream> <name> # 创建 Consumer
:consumer delete <stream> <name> # 删除 Consumer
:consumer info <stream> <name>   # 查看 Consumer 详情

# 消息操作
:publish <subject> <data>       # 发布消息
:msg get <stream> <seq>         # 获取消息
:msg delete <stream> <seq>      # 删除消息
:msg list <stream> [count]      # 列出消息

# KV 操作
:kv list                        # 列出 Bucket
:kv keys <bucket>               # 列出键
:kv get <bucket> <key>          # 获取值
:kv put <bucket> <key> <value>  # 设置值
:kv del <bucket> <key>          # 删除键

# 系统操作
:connect <url>                  # 连接服务器
:disconnect                     # 断开连接
:refresh                        # 刷新当前视图
:help [topic]                   # 查看帮助
:quit / :q                      # 退出应用
```

**命令特性：**
- Tab 自动补全命令和参数
- 支持命令别名（如 `:ls` 等同于 `:list`）
- 命令历史记录，可用 ↑/↓ 浏览
- 支持管道操作（如 `:stream list | grep orders`）

#### 4.4.3 高效导航设计

**快速跳转：**
- **数字 + g/G**：跳转到第 N 行（如 `5g` 跳转到第 5 行）
- **g{标记}**：跳转到标记位置（如 `gs` 跳转到 Streams 页面）
- **H / M / L**：跳转到当前视图的顶部/中间/底部

**标记功能（类 Vim marks）：**
- **m{a-z}**: 设置标记（如 `ma` 在当前位置设置标记 a）
- **'{a-z}**: 跳转到标记位置（如 `'a` 跳转到标记 a）
- 支持在不同页面间跳转标记

#### 4.4.4 强大的搜索和过滤

**搜索功能：**
- 支持正则表达式搜索
- 支持模糊搜索（fuzzy search）
- 搜索结果高亮显示
- 支持在搜索结果间跳转（n/N）

**过滤功能：**
- 多条件组合过滤
- 支持比较运算符（>、<、=、!=）
- 支持日期范围过滤
- 过滤条件可保存和复用

**示例：**
```
# 搜索名称包含 order 的 Stream
/order

# 过滤消息数大于 1000 的 Stream
:filter messages > 1000

# 过滤创建时间在 7 天内的 Consumer
:filter created > 7d
```

## 8. 技术难点和解决方案

### 8.1 异步操作处理
**难点**: Bubble Tea 是同步渲染框架,但 NATS 操作可能是异步的

**解决方案**:
- 使用 Bubble Tea 的 Cmd 机制进行异步操作
- 定义清晰的消息类型(Msg)来处理异步结果
- 在等待期间显示加载状态

```go
// 异步加载 Stream 列表
func loadStreams(client *nats.Client) tea.Cmd {
    return func() tea.Msg {
        streams, err := client.ListStreams()
        return StreamsLoadedMsg{Streams: streams, Err: err}
    }
}
```

### 8.2 大量数据展示
**难点**: Stream 可能包含大量消息,一次性加载会导致性能问题

**解决方案**:
- 实现分页或虚拟滚动
- 使用懒加载,只在需要时加载数据
- 提供搜索和过滤功能减少显示内容

### 8.3 状态管理
**难点**: 应用有多个页面和复杂的状态转换

**解决方案**:
- 采用单一状态树模式
- 使用清晰的页面状态枚举
- 实现状态持久化保存用户配置

### 8.4 错误处理
**难点**: NATS 操作可能失败,需要友好的错误提示

**解决方案**:
- 统一的错误类型定义
- 在 UI 上清晰显示错误信息
- 提供重试机制
- 记录错误日志供调试

```bash
# 或者使用 go install
go install go-nats-manager-demo
```

```bash
# 直接启动，显示炫酷启动画面
./freexnats

# 指定 NATS 服务器地址
```

支持通过配置文件或命令行参数配置连接信息:

```yaml
# ~/.freexnats/config.yaml
nats:
  url: "nats://localhost:4222"
  username: ""
  password: ""
  timeout: 10s
```

### 10.4 命令行参数
```bash
freexnats --url nats://localhost:4222 --user admin --pass password
freexnats --config ~/.freexnats/config.yaml
freexnats --no-splash    # 跳过启动画面
freexnats --version      # 显示版本信息
freexnats --help         # 显示帮助信息
```
